Теперь у меня полная картина библиотеки. Это **донорский движок** (бэктест + метрики + WF + warmup), а grid-оркестратор будет написан отдельно и будет импортировать эти utils. Составляю план с учётом этого контекста.

### 1.2. `time_utils.py` — `calculate_median_timedelta`: монотонность + защита от delta ≤ 0

**Файл:** `utils/time_utils.py`, функция `calculate_median_timedelta`

- Индекс `[t, t, t]` (все одинаковые) → median = 0 → `ValueError`
- Перемешанный индекс → `ValueError`
- Корректный монотонный индекс → ок

## ФАЗА 4: INVALID_METRIC_VALUE — контракт для grid

### 4.1. `constants.py` — добавить explicit контракт-комментарий

**Файл:** `utils/constants.py`, строки 14–21

**Что сделать:**
- Расширить комментарий:

```
# INVALID_METRIC_VALUE = -999.0
# CONTRACT for grid consumers:
#   Any trial with objective metric == INVALID_METRIC_VALUE MUST be excluded
#   from ranking. Do NOT sort/argmax without filtering.
#   Suggested filter: result.metrics[objective] > INVALID_METRIC_VALUE + 1.0
#
# MAX_VALID_METRIC = 9999.0  
# CONTRACT: Any metric > MAX_VALID_METRIC is treated as outlier / numerical artifact.
#   Suggested filter: result.metrics[objective] < MAX_VALID_METRIC
```

**Почему для grid критично:**
Grid **не** использует Optuna pruning/objectives. Grid-скрипт собирает все `BacktestResult.metrics[objective]` в список/DataFrame и делает `sort_values`. Без явного контракта разработчик grid может забыть отфильтровать `-999` → sentinel окажется на дне (при `max()` — ок), но при `min()` (Drawdown, risk-adjusted) — будет выбран **как лучший**. Контракт должен быть **в коде**, не только в голове.

---

### 4.2. Рассмотреть: перейти с sentinel `-999.0` на `float('nan')`

**Это рекомендация**, не обязательный фикс. Описание:
- `np.nan` естественно исключается из `np.nanmean`, `np.nanmax`, pandas `sort_values(na_position='last')`
- Grid-скрипт на pandas будет автоматически **игнорировать** невалидные метрики
- Минус: нужен рефактор **всех** мест, где `== INVALID_METRIC_VALUE` → `np.isnan`

**Решение**: оставить на этапе grid-написания. Если sentinel-подход уже работает во всём проекте — не ломать; но добавить helper:

```python
def is_valid_metric(value: float) -> bool:
    return np.isfinite(value) and value > INVALID_METRIC_VALUE + 1.0 and value < MAX_VALID_METRIC
```

Добавить в `utils/constants.py` или в отдельный `utils/metric_helpers.py`.
